How to review

What to look for in the Designer

The client has login access to the Webflow project. Signed in, the following four checks verify that every deliverable above is in place.

Inspection checklist

1
Navigator panel (left). Eight named sections under Body — hero-section through cta-band — plus three green Component icons: nav-marketing, nav-profile, footer-global.
2
Style panel (right). Click any section; the class name and correct background color appear under Style. Backgrounds match the brand palette.
3
Variables panel (left sidebar, V icon). Base collection shows all 11 color variables with hex values matching the brand guide.
4
Site Settings → Fonts. DM Serif Display and DM Sans are loaded with the weight sets used across the design system.
Note
The original Week 1 checklist included a step for global custom head code (smooth scroll, reduced-motion). That step was removed because Webflow's built-in Interactions respect prefers-reduced-motion natively, and smooth anchor scrolling is a native setting. No custom code is needed anywhere on this project, matching the stated no-code constraint.

Phase 2 — core pages

With the foundation in place, Phase 2 populates the landing page with full content from the design (hero, trust bar, how-it-works, philosophy cards, testimonials, CTA band, waitlist form embed), builds the Partner Profile CMS template with all AT-5 fields, configures one-way Whalesync from Airtable to the Webflow CMS, and stands up the filterable directory page.

All Phase 2 work stays within the same stack: Webflow + Airtable + Whalesync + Typeform embeds. Zero custom code.
